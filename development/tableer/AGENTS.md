# Tableer development sandbox — правила агента

Этот каталог — внешний рабочий корень агента для Tableer. Целевой продукт
находится строго по пути `../../projects/tableer`, заданному в `sandbox.json`.
Tableer не содержит агента, genetic runtime или память: в нём только обычные
продуктовые модули.

## Что прочитать

1. `README.md`, `sandbox.json` и `TARGET.md`.
2. `memory/STATUS.md`, если песочница уже инициализирована.
3. `memory/pending.json`, если статус сообщает ожидающее действие.
4. Релевантную задачу из `tasks/` и файлы цели.

## Границы

- Изменяй продуктовый код только в явной цели `../../projects/tableer`.
- `memory/` принадлежит genetic runtime. Агент вручную пишет там только
  `response.json` по шаблону текущего pending action.
- Не переноси agent code, memory, инструкции разработки или runtime в Tableer.
- Не расширяй targetDirectory на другие проекты без отдельного решения.
- Это архитектурная песочница разработки, не OS security sandbox.

## Цикл одной законченной правки

1. Выполни `npm run status`; если памяти нет — один раз `npm run setup`.
2. Выполни одну законченную правку в Tableer и его продуктовую проверку.
3. Выполни `npm run scan` из этого каталога.
4. Если появился `memory/pending.json`, заполни `memory/response.json` честным
   конкретным выводом и выполни `npm run resolve` до состояния ready.
5. Не выдумывай инструкцию: для слабой закономерности выбери `discover/skip`.
6. Запиши в evidence только проверяемый результат, а не обещание эффекта.
