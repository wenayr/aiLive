# Основа проекта

## Дальняя точка

Конечный вектор — динамический проект, который меняет форму под намерение
пользователя: голосовой интерфейс может стать чатом, рабочей панелью или иным
продуктом, а генеративные слои со временем переходят от сборки обычного кода к
динамическому проектированию и компиляции. Сейчас эта точка недостижима; она
задаёт направление, но не оправдывает преждевременную инфраструктуру.

## Ближайшая проверяемая гипотеза

Обычный небольшой проект разрабатывается как обычно. Рядом работает маленький
генетический модуль. После сохранённой правки он:

1. замечает только изменившиеся наблюдаемые файлы;
2. выбирает относящийся к ним вектор проверки;
3. просит агента сделать одно предметное заключение;
4. сохраняет одобренную инструкцию, исключение, историю и счётчики;
5. при следующей похожей правке напоминает именно эту инструкцию.

Первый эксперимент должен ответить не на вопрос «можно ли построить всю систему
будущего», а на вопрос: снижает ли такой локальный цикл число повторных ошибок и
стоимость следующей похожей задачи при приемлемых накладных расходах.

## Архитектура как песочница

Ответственный модуль сам является песочницей: у него есть каталог, инструкция,
ограниченные входы, собственное состояние, локальные проверки и наблюдаемые
результаты. Сложный проект может быть композицией таких песочниц. Это название
архитектурной границы разработки, а не обещание изоляции недоверенного кода на
уровне операционной системы.

## Ограничение текущего среза

Никакой встроенный API модели не нужен. Codex уже открыт внутри каталога и сам
выполняет задачу, читает запрос генетического модуля и записывает ответ. Git
фиксирует стабильный рецепт и отобранные выводы, но commit не является триггером
модуля: ошибка должна быть замечена до того, как агент успеет построить на ней
следующий костыль.
