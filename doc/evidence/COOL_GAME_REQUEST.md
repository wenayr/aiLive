# Один сырой запрос в две ветки

Исходная пользовательская фраза была одинаковой: «Сделай игру классной».

| Ветка | Самостоятельное сужение | Почему |
| --- | --- | --- |
| Simple | Турбо-рывок по `Shift`, cooldown и выхлоп | Самый короткий заметный эффект в одном существующем runtime |
| Sandbox | Map-derived energy cores, которые дают turbo burst | Использует уже зрелый arena contract и проверяет активную instruction без нового generator |

Это не попытка доказать, что одна интерпретация «правильнее». Ценность опыта —
в том, чтобы у каждой ветки был явный, ограниченный и проверяемый выбор вместо
бесконечного раскрытия слова «классной».

## Фактический результат

- Direct-ветка добавила turbo burst, cooldown и выхлоп, не создав новой
  продуктовой границы.
- Sandbox-ветка расширила только `createArena` и `createGame`: arena выдаёт три
  безопасных `corePads`, game превращает их в collectable state и заряд turbo.
  Renderer и browser binding лишь проецируют это состояние.
- Product tests подтвердили безопасность core pads и преобразование одного core
  в один turbo burst; обе страницы загружены в браузере без ошибок консоли.
- Genetic state увидел два изменённых watched paths и вызвал существующий
  `reviewer/inspect`, а не новый `discover`. Verdict — `clear`; у инструкции
  `generator-contract-runtime-boundary` теперь `approvedChecks: 1`.

Это первый полезный сигнал: накопленная инструкция не заставила строить новый
слой, а сузила проверку после того, как sandbox сам выбрал ограниченную фичу.
Один такой clear всё ещё не доказывает экономию усилий; он лишь доказывает, что
следующая похожая правка прошла через подходящую узкую проверку.
