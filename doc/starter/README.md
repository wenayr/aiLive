# Практический starter-kit

Этот каталог превращает идею живых модулей в проверяемый рабочий процесс. Он не является реализацией автономной генетической базы и не обещает автоматически разобрать любой проект. Его задача — дать единый формат наблюдения, экспериментов и накопления evidence.

## Быстрый старт

1. Передайте агенту [SESSION_BOOTSTRAP.md](./prompts/SESSION_BOOTSTRAP.md).
2. Выберите реальный vertical slice из [первой кампании](./experiments/FIRST_CAMPAIGN.md).
3. Выполняйте по одному режиму из [операционного router](./prompts/README.md).
4. Следуйте [Runbook первых семи сессий](./RUNBOOK.md).
5. Оценивайте результат по [acceptance matrix](./experiments/ACCEPTANCE_MATRIX.md).

## Что является успехом

Успехом считается не количество созданных пакетов или MCP-серверов, а доказанное улучшение следующей похожей задачи:

- агент читает меньше нерелевантного контекста;
- изменение затрагивает меньше независимых частей;
- результат быстрее проверяется локально;
- повторное применение требует тонкого project binding;
- generator создаёт проверяемое семейство результатов;
- неудачный candidate можно безопасно отклонить или встроить обратно.

## Состав

- `standards/` — слои и фасады, зрелость, guard/evidence и каталог;
- `templates/` — карты проекта, слоя, module candidate, genome и generator candidate;
- `prompts/` — bootstrap и короткие инструкции отдельных режимов;
- `experiments/` — первая сложная кампания и единая матрица оценки.

## Ограничение первой версии

Генетический модуль пока следует считать исследовательской ролью, а не установленным фактом. Его предложения являются гипотезами. Решения `EXTRACT`, `FUSE`, `GENERATE` и `EXPOSE_MCP` получают силу только после materialization в candidate scope и независимой проверки.
