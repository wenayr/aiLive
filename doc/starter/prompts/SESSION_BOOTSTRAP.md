# Bootstrap новой рабочей сессии

Ты работаешь над исследовательской системой живых модулей.

Сначала прочитай:

1. `doc/concept/CURRENT_MODEL.md`;
2. `doc/genetic-module/REQUIREMENTS.md`;
3. `doc/starter/prompts/README.md`;
4. только тот стандарт и тот режим, которые нужны текущей задаче.

Не загружай полный v5 и весь каталог instructions без необходимости.

Перед действием объяви один режим: `BUILD`, `SCAN`, `GENERATOR_HUNT`, `ACQUIRE`, `EXTRACT` или `REVIEW`. Если запрос смешивает режимы, раздели его на последовательные сессии и выполни только текущую.

Для любого архитектурного утверждения различай:

- наблюдаемый факт;
- интерпретацию;
- рекомендацию;
- неизвестное.

`NO_CHANGE`, `KEEP_LOCAL` и `NO_GENERATOR` являются допустимыми результатами. Не оптимизируй количество модулей или MCP.

Если режим создаёт candidate, пиши только в разрешённый candidate scope и подготовь evidence/rollback. Если режим оценивает candidate, не продолжай его замысел автоматически и не подтверждай его собственной самооценкой.

В конце верни:

1. режим и scope;
2. созданные артефакты;
3. выполненные проверки;
4. unresolved risks;
5. рекомендуемый следующий режим.
