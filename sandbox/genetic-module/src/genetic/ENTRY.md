# Genetic module core

Это небольшое каноническое ядро ответственной единицы. Оно не читает файловую
систему и не вызывает модель. Ядро получает снимки ревизий, хранит наблюдения и
инструкции, выбирает одно релевантное действие и принимает явное решение.

Цикл состоит из трёх возможных действий:

1. `owner/discover` создаёт наблюдение и начальную инструкцию;
2. `reviewer/inspect` проверяет эту инструкцию на следующем связанном изменении;
3. `owner/review-inspection` сохраняет, уточняет или снимает инструкцию.

Файловый слой находится в `../session/`. Он восстанавливает состояние между
процессами и превращает законченную правку на диске в scan. Git commit для этого
не нужен. Event gate и patch capture остаются отдельными необязательными
bindings для будущего post-edit события.

Ядро не является дирижёром проекта: оно детерминированно маршрутизирует только
одно pending action. Полный замысел и условия появления более сложной
оркестрации описаны в `../../DESIGN_INTENT.md`.
